# Image Asset Role Rules（图片资产角色判定）

> 本文件承接 `IMAGE_PIPELINE.md` 的参考图 / 上传图规则。  
> 目标是让 Agent 先判断图片在本轮中的角色，再决定 `text-image` / `image-image` / `image-text` 和 edges，而不是看到图片就机械保留主体。

## 一、角色类型

| 角色 | 判断信号 | 默认处理 |
|------|---------|---------|
| 创作主体 | “用这张产品图做主图”“以这个角色为主角”“保留这个 Logo” | `image-image + editAction:redraw`，edge 指向源图 |
| 待编辑素材 | “在这张图上改”“给他戴帽子”“换衣服”“局部修改”“保持其他不变” | `image-image + editAction:redraw`，prompt 写“保留 X，仅改 Y” |
| 风格参考 | “参考这个风格 / 画风 / 质感 / 色调”“类似这张的感觉” | 可用 `text-image` 或 `image-image`，但不得承诺主体还原 |
| 色彩参考 | “按这个配色”“用这张的色调” | prompt 提取色彩关系，不保留主体 |
| 构图参考 | “按这个构图 / 版式 / 视角” | prompt 描述构图关系，不保留主体，除非用户明确要求 |
| 内容参考 | “里面这种元素 / 氛围 / 场景” | prompt 抽取可复用内容，不保留具体主体 |
| 已有节点上下文 | 消息含真实 `node-...` 或画布快照提供节点 | 下游 edge.source 必须用真实节点 ID |
| 反推对象 | “分析这张图”“反推 prompt”“照着这个做一段描述” | `image-text`，edge 指向源图 |

## 二、判定优先级

```text
用户明确修改/保留指令
  > 真实 node 上下文
  > 领域 Skill 约束（如电商产品还原、Logo 扇出物料）
  > 用户对参考图用途的描述
  > 图片内容本身
  > 默认追问
```

## 三、必须追问的情况

只在以下情况追问，且最多一轮：

1. 用户上传了图片但完全没说用途：问“你是想保留主体修改，还是只参考风格 / 构图？”
2. 用户同时要求严格保留主体，又要求大幅改变视角、姿态、结构或构图。
3. 多张参考图角色冲突：无法判断哪张是主体、哪张是风格。
4. 反推 / 复刻 / 编辑意图混在一起，直接执行会选错 subType。

## 四、subType 决策

| 场景 | subType | 必填 |
|------|---------|------|
| 无参考图的新图 | `text-image` | `content` |
| 保留 / 修改 / 基于已有图重绘 | `image-image` | `editAction:redraw` + `edges[]` |
| 图片反推文本 | `image-text` | `edges[]` |
| Logo / 产品 / 角色多图扇出 | `image-image` | 所有扇出节点指向同一 anchor |
| 只参考风格且不保留主体 | 可 `text-image` | prompt 写明风格来源，不写“保留主体” |

## 五、Prompt 口径

### 5.1 待编辑素材

使用“保留 + 仅改”的结构：

```text
基于参考图进行重绘，保留[主体身份 / 外观 / 构图 / 背景 / 画风]，
仅将[变化点]修改为[目标变化]，其余内容保持一致。
```

禁止写成从零生成整张图。

### 5.2 风格参考

写“提取风格，不保留主体”：

```text
参考上传图的[色彩关系 / 光影质感 / 材质表现 / 构图节奏]，
重新生成[新主体]，不复刻参考图中的具体人物 / 商品 / Logo。
```

### 5.3 产品 / Logo / 角色锚点

多图扇出时必须显式声明：

```text
保留源图中主体的全部外观特征（Logo / 配色 / 外形 / 结构 / 工艺细节），
仅修改背景 / 构图 / 文案区 / 使用场景。
```

## 六、Engineer 交接要求

有真实源节点时，交接给 `engineer` 的 Prompt Pack 必须包含：

- `edges[]`
- `sourceNodeId`（冗余锚点，跨轮节点必填）
- `editAction:redraw`（`image-image` 必填）
- `agentNodeType` 与节点语义一致

不得使用 `<anchor>`、`<#1>`、`node_product_final` 等语义别名冒充真实节点 ID。

## 七、自检

输出 Prompt Pack 前检查：

- [ ] 是否已判断每张上传图 / 画布图的角色？
- [ ] 需要保留主体的节点是否全部 `image-image`？
- [ ] `image-image` 是否都有 `editAction:redraw`？
- [ ] 跨轮引用是否使用真实 `node-...`？
- [ ] prompt 是否区分“保留主体”和“参考风格”？
- [ ] 多图扇出是否围绕同一 anchor，而不是独立 `text-image`？
