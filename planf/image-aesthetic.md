# Image Aesthetic Core（图片质量心智 · 共享护栏）

> 本文件是所有图片产出 skill（`prompter` / `ecom-image` / `ecom-ref-gen` / `brand-designer` / `brochure` / `storyboard-master` 等）的共享质量护栏。
>
> 它**不**定义输入输出协议（那是 `prompter-core.md` 的事），**只**定义"画面质量心智"——8 维度框架、抽象→视觉翻译、反廉价感清单、默认审美增强、多图多样性。
>
> 任何写图片 prompt 的 skill **必须**引用本文件，并在 Post-check 中校验。

## 一、四项基础原则

每次创作必须同时满足：

1. **意图优先，审美增强**——审美提升只能在用户意图允许的范围内发生；不擅自替换主体、风格、文案、比例
2. **专业代劳，减少负担**——能推断就推断，能默认就默认，能直接生成就生成；禁止把"整理 prompt / 设计方案 / 创作逻辑"推给用户
3. **结果导向，不堆形容词**——prompt 服务于最终画面质量，不服务于语言表面；抽象词必须有具体画面控制兜底
4. **默认高标准**——除非用户明确要求低完成度风格（手绘草图、像素风、儿童涂鸦等），始终追求成熟、统一、完成度高的视觉结果

## 二、抽象词 → 具体画面翻译表（强制）

抽象审美词**禁止单独使用**，必须同时给出对应的具体画面控制：

| 抽象词 | 翻译为具体画面控制 |
|--------|------------------|
| 高级感 | 色彩克制（≤3 色）、留白合理、材质统一、光线收敛、构图简洁 |
| 电影感 | 单侧主光、高光比、色彩分级、空间层次、环境氛围保留 |
| 氛围感 | 光线方向、色温、背景虚化程度、前景遮挡关系 |
| 有质感 | 具体材质的边缘高光、纹理层次、表面反射特征 |
| 商业成片感 | 主体突出、信息层级清晰、缩略图识别度高、背景干净 |
| 艺术感 | 构图张力、形式节奏、色彩控制方式、留白比例 |
| 极简 | 色彩数量 ≤3、背景纯净、元素数量受控、留白比例明确 |
| 科技感 | 冷色主调（蓝青紫）、几何感构成、硬光与点光、局部反射高光带、负空间 |
| 复古感 | 偏黄/偏暖偏低饱、颗粒/光晕/染色处理、经典字体气质、做旧边缘 |
| 国潮感 | 中国传统色（朱砂/青花/茶白）、水墨/工笔元素、对称构图、传统装饰性纹样 |
| 治愈感 | 自然光、暖白/淡粉/淡青色调、柔焦或浅景深、画面留白与负空间 |

**原则**：所有抽象形容词必须落地到主体、场景、构图、光影、色彩、材质、背景、用途适配 8 维度中至少 2 个。

## 三、8 维度核心框架

8 维度是完整的画面控制框架，但**不要求每次等权覆盖**。应根据任务类型判断哪些维度是重点，哪些可以简化或省略。

| 维度 | 控制内容 |
|------|---------|
| **主体** | 是什么、数量、主次关系、外观特征、姿态朝向、镜头距离、画面占比、位置 |
| **场景** | 空间类型、前中后景层次、环境元素与主体关系、空间纵深 / 透视 / 虚化 |
| **构图** | 横 / 竖 / 方、视觉重心、留白位置、疏密关系、视线引导路径 |
| **光影** | 主光方向、辅光作用、硬 / 柔光、光比强弱、阴影清晰度、高光控制 |
| **色彩** | 主色调、辅色、点缀色、冷暖关系、饱和度策略、色彩数量控制 |
| **材质** | 材质种类、质感重点、反射 / 粗糙度、边缘高光、微表面细节 |
| **背景** | 如何衬托主体、是否承接文案、空间逻辑、完成度 |
| **用途适配** | 海报留文案区、电商产品识别清晰、封面缩略图识别度、社媒第一眼吸引力 |

### 3.1 按任务类型的维度权重参考

| 任务类型 | **重点维度**（必须有具体控制描述） | 可简化维度（信息充足时简要带过） |
|----------|----------------------------------|------------------------------|
| 产品 / 电商图 | 主体、材质、光影、背景 | 场景纵深、用途适配 |
| 人像写真 | 主体、光影、色彩 | 材质细节、用途适配 |
| 海报 / 封面 | 构图、用途适配、色彩 | 材质、场景纵深 |
| 概念场景 / 插画 | 场景、构图、色彩、光影 | 材质、用途适配 |
| Logo / 品牌标志 | 主体、色彩、背景 | 场景、光影 |
| 生活方式 / 氛围图 | 场景、光影、色彩 | 材质细节、用途适配 |
| 角色设定图 | 主体、构图、色彩 | 场景、光影、材质 |
| 分镜板 | 主体、构图、场景 | 材质、用途适配 |

**使用原则**：重点维度**必须**有具体控制描述；可简化维度在信息充足时简要带过，不强制展开。

## 四、Prompt 写"如何实现"，不只写"要什么"

| 不推荐（要什么） | 推荐（如何实现） |
|---------------|-----------------|
| 背景高级 | 背景采用低饱和深灰蓝渐变，中部保持干净衬托主体，侧边留可用于文案的留白区 |
| 光影有电影感 | 单侧主光形成明确明暗面，弱轮廓光分离主体与背景，保留阴影层次，光比偏高，高光克制 |
| 很有质感 | 金属表面有清晰受控的高光条带，玻璃边缘有透亮折射，织物保留细密纹理与自然褶皱 |
| 商务感强 | 深蓝与银灰双配色，整体偏低饱和，构图对称稳定，纵向居中，负空间均衡 |
| 适合小尺寸辨认（Logo） | 厚实几何块面构成，外轮廓完整闭合，内部仅保留一个关键负形识别点 |
| 颜色舒服 | 主色冷蓝（约 #2A5C8F）+ 辅色暖白 + 点缀色琥珀黄，色相控制在 3 个以内，饱和度统一偏低 |

**原则**：prompt 描述的是"图形怎么长、文字怎么改、轮廓怎么收、负形怎么留、背景怎么控"，不是"它应该满足什么用途"。

## 五、主动规避常见廉价感（反模式清单）

写 prompt 时主动规避，Post-check 时主动检查：

- ❌ 主体不突出，视觉中心模糊
- ❌ 背景随意拼凑、脏乱、无空间逻辑
- ❌ 光线混乱，没有主光方向
- ❌ 色彩高饱和乱堆、冷暖失控
- ❌ 材质塑料感、皮肤蜡感
- ❌ 五官、手部、结构、透视不稳定
- ❌ 特效滥用，烟雾、粒子、光斑无意义堆砌
- ❌ 海报留白不足，文案区被挤占
- ❌ 电商图产品不够大、不够清楚
- ❌ Logo 主体悬浮于复杂摄影背景上、被场景喧宾夺主
- ❌ 文字看起来像"默认字体直接贴上去"，缺设计感
- ❌ 抽象词堆砌（"高级、有质感、商务、国际化"）但没有具体画面落地

## 六、默认高审美增强策略

在不违背用户意图的前提下，**默认增强**以下维度（用户明确要求低完成度风格时关闭）：

| 维度 | 具体增强方向 |
|------|------------|
| **构图** | 建立明确视觉重心，控制主体占比，强化前中后景关系，为文案留白，让视线流动清晰 |
| **背景** | 服务主体，即使极简也有层次，与主体色彩 / 材质 / 结构形成呼应 |
| **光影** | 有方向性主光，强化主体体积感，轮廓清晰，控制高光不过曝，阴影有层次 |
| **色彩** | 主色 + 辅色 + 点缀色结构，控制色彩数量，根据用途调整冷暖，保证整体统一 |
| **材质** | 对关键材质重点刻画边缘高光、纹理层次、表面反射，保持不同材质间差异化 |
| **完成度** | 五官稳定、手部正常、透视准确、边缘干净、接触关系真实、反射阴影符合物理逻辑 |

## 七、多图多样性规则

### 7.1 默认开启差异化

当生成多张图时（默认 4 张），**每张必须在方案与 prompt 层面都具有差异化**，给用户提供有区分度的选择：

- 构图视角（特写 / 中景 / 全景；正面 / 侧面 / 俯仰角）
- 光影方案（主光方向、光比、冷暖、时段氛围）
- 色彩基调（主色调、饱和度、冷暖关系）
- 场景氛围（环境元素、背景层次、空间关系）
- 风格倾向（写实 / 插画 / 概念，**在不违背用户意图前提下**）
- 主体姿态、动作或表现形式

### 7.2 关闭差异化的情况

以下任一情况下，**不强制差异化**，应严格围绕用户指定方向生成，多张图之间只允许细节级微调：

1. **用户 prompt 高度具体明确**——已精细到结构、风格、构图、内容、配色等关键细节
2. **存在指向性参考图**——上传图意图明确（保留主体 / 复现构图 / 沿用风格）
3. **领域 skill 有额外约束**——如 ecom-ref-gen 的"产品 100% 还原" / brand-designer 的 Logo 一致性

### 7.3 ratio 不属于差异化维度

**铁律**：除非用户明确要求多比例方案，或子任务明确要求做尺寸适配，否则**同批次所有结果必须保持同一 ratio**。多样性只在内容、构图、镜头、光影、色彩、场景、动作维度体现，**不得通过改变 ratio 来实现**。

### 7.4 判断优先级

```
用户具体需求 > 参考图 > 领域 skill 约束 > 通用多样性规则
```

## 八、默认创作参数

| 参数 | 默认值 | 例外条件 |
|------|--------|---------|
| 分辨率 | 4K | 内容简单、无小人脸 / 文字 / 精细细节，且 2K 不影响质量时可用 2K |
| 生成张数 | 4 张（默认追求多样性） | 用户明确指定数字（如"生成 8 张"、"生成 1 张精修") 时按用户要求 |
| 画幅比例 | 按用途推断 | 手机竖版 → 9:16；常规海报 / 人物 → 3:4；方图 → 1:1；横版场景 → 16:9 |
| 并行生成 | 是 | — |
| 风格 | 根据用途和子任务自动推断 | — |

> 画幅比例属于**批次级固定参数**。同一轮生成的多张图，若用户未明确要求不同尺寸 / 不同比例适配，则所有图必须使用同一个 ratio。

## 九、用户要求保留规则

用户明确提出的所有创作要求**必须完整保留**至最终 prompt，包括但不限于：

- 主体、场景、风格、构图、色调、材质、服装外观
- 文案文字（一字不改）、比例、用途
- 参考图要求、保留项、禁止项

生成前必须检查：是否已覆盖用户所有明确要求。**若需求间存在冲突**，先指出并让用户以最简方式确定优先级，**不得擅自取舍**。

## 十、参考图使用规则

### 10.1 用途必须先确认

用户上传图片时，必须先确认用途（如未说明）：

- "这张图你是想保留人物，还是主要参考风格？"
- "你是想在原图上修改，还是参考它重新生成？"

### 10.2 有任意图片输入时

**必须使用 `image-image`，禁止用 `text-image` 重新生成**。

当用户要求"同时保留原图主体 + 大幅改变视角 / 姿态 / 构图"时，须提示冲突并请用户确认优先级。

### 10.3 引用方式

- 画布已有节点：用 `from: <真实节点 ID>` 锚定，加 `edges` 入边指向新节点
- 用户上传图：在 prompt 自然语言里描述为"上传的参考图 / 第一张图"，禁止暴露 `upload_image_*` 等内部文件名

## 十一、多轮迭代规则

- **继承已确认的有效信息**，只针对变化点调整
- **将抽象修改意见翻译为具体画面控制细则**（"再高级一点" → 具体到色彩克制 / 留白增加 / 光影收敛等）
- **禁止**重新追问全部参数
- **禁止**简单重复上一版

生成后简洁引导用户继续迭代（提示 2~4 个方向，例如：更写实 / 更电影感 / 放大主体 / 优化光影 / 提升清晰度 / 基于某张精修）。

## 十二、Post-check（所有图片 skill 必跑）

任何图片产出 skill 在输出 Prompt Pack 之前，必须在 thinking 中跑一遍：

- [ ] **重点维度覆盖**：按任务类型，重点维度（§ 3.1）是否都有具体控制描述？
- [ ] **抽象词翻译**：是否还残留未翻译的抽象审美词（高级感 / 电影感 / 氛围感 / 质感 / 商业成片感等）？
- [ ] **写"如何"而非"什么"**：prompt 是否描述具体视觉实现方式，而非要求口号？
- [ ] **反廉价感**：§ 五的 12 条反模式是否主动规避？
- [ ] **多图差异化**：多图生成时，是否在 § 7.1 的 6 维度上有显式差异化？或者命中 § 7.2 关闭条件？
- [ ] **同 ratio 铁律**：同批次多图 ratio 是否一致？
- [ ] **用户明确要求保留**：用户的主体、风格、文案、比例、保留项、禁止项是否完整保留？
- [ ] **参考图规则**：有图片输入时是否用 `image-image` + `from`？

## 十三、领域 skill 引用本文件的标准方式

各领域 skill 应在文档开头加一句：

```markdown
> 共享规范：
> - 图片质量心智见 `skills/_shared/image-aesthetic.md`
```

并在 Post-check 章节加一句：

```markdown
> 通用图片画面质量自检见 `skills/_shared/image-aesthetic.md § 十二`，本节只列**领域专属**的额外校验。
```

不得复制本文件的规则到自己的 SKILL.md（避免规则漂移）。

## 十四、与其他共享文件的边界

| 共享文件 | 职责 | 与本文件的关系 |
|---------|------|--------------|
| `prompter-core.md` | C 阶段共享骨架（Contract / Prompt Pack 格式 / 通用字段约束 / 通用 Post-check） | 工程契约层；本文件是质量心智层。两者**正交**，互相引用不重复 |
| `asset-consistency.md` | 跨节点资产一致性规则（角色锚定描述 / from 字段 / sourceNodeId） | 一致性是工程层；本文件管美学。一致性优先级 ≥ 多图多样性 |
| `handoff-brief.md` | Skill 之间的 brief 传递格式 | 协议层；本文件不涉及 |
| `domain-baseline.md` | 各领域的安全 baseline | 领域兜底；本文件是通用质量护栏 |
| `state-header.md` | thinking 中的 State Header 格式 | 状态协议；本文件不涉及 |

**优先级**：用户明确需求 > 安全红线 > 资产一致性（asset-consistency）> 领域 skill 约束 > 本文件（image-aesthetic）> 默认策略
