# Asset Consistency Rules

> 跨片段/跨页面的资产一致性规则。适用于多 Clip 视频、漫剧连续多话、品牌多触点设计等需要跨节点保持一致性的场景。

## 一、什么是资产一致性

当一个创意任务包含多个生成节点（多张图 / 多段视频 / 多页漫画），且这些节点之间需要保持人物、场景或道具的视觉一致性时，就需要资产一致性管理。

## 二、资产注册表的职责

资产注册表（`assetRegistry`）是文档链最终阶段输出的一部分，它的职责是：

1. 为每个跨节点出现的角色/场景/道具建立**唯一的文字锚定描述**
2. 记录每个资产出现在哪些 Clip / 页面 / 节点中
3. 传递给 `prompter` / `video-prompt`，确保生成提示词中包含一致的描述

## 二.A、主体锚点契约（Character / Model / Product Anchor）

当用户明确把上传图或画布节点绑定为主体时，该素材不再是普通参考图，而是**主体锚点**。典型信号：

- `图片1 女主` / `图片2 男主` / `这张是主角` / `这个人后面都用他`
- `这是产品` / `这是商品` / `这是主推款` / `产品按这张来`
- `这是模特` / `这个模特出镜` / `主播用这个人`

主体锚点在 `assetRegistry` 中必须保留以下字段：

| 字段 | 必填 | 说明 |
|---|---|---|
| `name` | 是 | 稳定资产名，如 `贞贞` / `男主` / `除螨仪主推款` |
| `kind` | 是 | `character` / `model` / `product` / `prop` |
| `provenance` | 是 | 来自用户明确主体绑定时写 `user_subject` |
| `sourceNodeId` | 条件必填 | 主体来自上传图或画布节点时必须是真实 `node-...` |
| `sourceLabel` | 建议 | 人类可读来源，如 `图片1` / `上传产品图` |
| `coreVersion` | 是 | 主体稳定外观描述，后续主体卡 / 变体 / 视频 prompt 都继承 |
| `clips` | 是 | 出现在哪些 Clip / 页面 / 图位 |

规则：

- `sourceLabel` 只是可读说明，**不能**替代 `sourceNodeId`。
- 没有真实节点来源、仅用户文字描述的主体，不写 `user_subject`；应写 `to_generate`，先生成主体卡。
- 跨 2 个及以上 Clip / 页面 / 图位出现的人物、模特、产品，必须进入 `assetRegistry`，不得只在剧本文字里反复称呼。
- 产品和出镜人物 / 模特同级处理：营销视频中产品需要一致，人物脸、身形、服装锚点同样需要一致。

## 二.B、主体卡与变体传递字段

主体卡（anchor card）是后续变体和视频引用的母本。变体（variant）必须显式继承主体卡，不能只靠文字记忆。

主体卡字段：

| 字段 | 必填 | 说明 |
|---|---|---|
| `anchorNodeId` | 生成后必填 | 主体卡 finished 后的真实 `node-...` |
| `anchorStatus` | 生成后必填 | 必须是 `finished` / `success` 才能进入变体或视频 |
| `anchorOutputUrl` | 生成后必填 | 可被下游引用的真实输出 URL |
| `anchorType` | 是 | `character-card` / `model-card` / `product-anchor` |

变体字段：

| 字段 | 必填 | 说明 |
|---|---|---|
| `variantOf` | 是 | 指向主体资产名，如 `贞贞` |
| `baseAssetId` | 是 | 主体卡真实 `node-...`，不能是 `图片1` / 临时 ID |
| `variantType` | 是 | 如 `雨夜湿发版` / `博物馆落泪版` / `手持产品版` |
| `changeSet` | 是 | 只列本变体相对主体卡改变的点 |
| `reusableForClips` | 是 | 明确适用的 Clip / 页面 / 图位 |
| `inheritLocked` | 是 | 必须列出保持不变的主体特征，如脸型、发型、体型、产品外观、Logo |

规则：

- 变体必须基于 `baseAssetId` 生成，禁止直接从风格图或普通文案生成。
- 多个变体必须并列继承同一主体卡，禁止“变体套变体”。
- `video-prompt` 引用资产时，必须能判断该 Clip 应引用主体卡还是某个 `variantType`。
- 变体没有 `baseAssetId` / `reusableForClips` 时，不得进入最终视频 workflow。

## 三、角色一致性规则

### 3.1 必须锚定的维度

- 年龄段 + 性别 + 民族面孔
- 发型（长度/颜色/风格）
- 体型
- 服装（具体款式和颜色，精确到上衣/下装）
- 标志性特征（如果有）

### 3.2 跨 Clip 约束

- 角色在所有 Clip 中的锚定描述必须一致
- 如果角色在某个场景中换装，必须在对应 Clip 的描述中明确说明
- 不允许在不同 Clip 的提示词中对同一角色使用矛盾的外貌描述

### 3.3 情绪与表情

- 角色在不同 Clip 中的情绪可以变化
- 但外貌锚定（发型/服装/体型）不随情绪变化
- 情绪描述用括号标注在角色名后：如"（烦躁的）女主"

## 四、场景一致性规则

### 4.1 必须锚定的维度

- 空间类型（卧室/客厅/办公室等）
- 墙面/地面材质和颜色
- 关键家具/摆设
- 窗户朝向和采光
- 整体色调和氛围

### 4.2 跨 Clip 约束

- 同一场景在所有 Clip 中的空间描述必须一致
- 不同场景之间的差异要足够明显
- 如果涉及同一场景的不同时段（如白天 vs 夜晚），光照可以变化但空间布局不变

## 五、道具一致性规则

### 5.1 必须锚定的维度

- 材质和颜色
- 尺寸和形状
- 关键细节（如品牌标识、独特造型）

### 5.2 道具分类

| 类别 | 说明 | 一致性要求 |
|------|------|----------|
| 产品（核心道具） | 用户上传或明确描述的产品 | 必须严格一致 |
| 静态陈设 | 场景中的固定摆设 | 场景级一致 |
| 动态道具 | 角色使用的工具/设备 | Clip 级一致 |

## 六、提示词中的一致性实践

### 6.1 角色出现时

每次角色出现在提示词中，都应包含完整的锚定描述，而不是简写：

正确："穿着浅灰色棉质居家服的齐肩棕色直发女性，28岁左右，亚洲面孔"
错误："女主角"

### 6.2 场景出现时

每次场景出现在提示词中，都应包含关键空间锚定：

正确："现代简约卧室，米白色墙面，人字纹浅棕色木地板，窗边放置编织花篮"
错误："卧室"

### 6.3 产品出现时

每次产品出现在提示词中，都应包含材质和颜色锚定：

正确："奶茶色加厚雪尼尔面料窗帘，垂坠感强，褶皱整齐均匀"
错误："窗帘"

## 七、连贯性校验清单

文档链最终阶段输出前，应完成以下自检：

1. 每个角色在所有出现的 Clip 中，外貌描述是否一致？
2. 每个场景在所有出现的 Clip 中，空间描述是否一致？
3. 产品在所有 Clip 中的材质/颜色描述是否一致？
4. 角色是否在没有换装说明的情况下穿了不同衣服？
5. 场景是否在没有时间跳转说明的情况下改变了布局？
